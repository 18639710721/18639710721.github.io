# 魔法方法介绍

  • doc 用来查看类的说明文档
  • module 用来查看当前操作的类所在的模块
    一个.py 文件就是一个模块
    当执行模块是类所在模块时，执行结果为__main__. 否则，执行结果是类所在模块的名称
  • __class__查看当前对象的类
    类也是对象，Demo 是 type 类的对象，d 是 Demo 类的对象
  • dict 用来获取类或者实例的属性，返回值为字典
    对象.__dict__获取的是普通字段，只有普通字段存储在对象中，除普通字段以为的成员都存储在类中，所有通过类.__dict__来获取
  • del () 方法，析构方法。当由类创建的实例对象，被删除或者在内存中被释放，将会自动触发执行
    当代码全部执行完毕才自动触发__del__() ，如果需要提前触发，则需要通过 del 关键字，删除所有对象后触发__del__()
  • call () 方法用于将对象变成一个可调用的对象，也就是说，当一个类中__call__() 方法时，其实例化得到的对象便是可调用的 (callable)
    带 () 是调用 不带是引用，@property

  • new () 方法用于创建与返回一个对象。在类准备将自身实例化时调用
    1.new_() 和__init__() 的区别
    new__方法，它会构造一个对象并返回，init 接收构造的对象并对其进行初始化。
    2.init__ 与 new 的关系总结：作用方面，__new__负责构造对象，init 负责初始化对象；执行顺序方面，__new__先执行，init 后执行；返回值方面，__new__返回构造的对象，init 只能返回 None；
    3.__init__是实例方法，new__是静态方法
    4. 返回对象自动触发__init     () 方法，init () 方法当中的 self，实际上就是__new__返回的实例，就是该对象。

  • 承一些不可变的 class 时 (比如 int, str, tuple)，__new__方法提供了自定义这些类的实例化过程的途径。

  • 应用：控制实例个数并
